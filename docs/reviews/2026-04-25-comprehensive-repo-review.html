0. TL;DR

Serious, differentiated local-first knowledge-base system with deep implementation and a large test suite. The product idea is genuinely novel (compile-not-retrieve, MCP-native, Obsidian-native, trust-aware). The blocker is not capability — it is product contract, dependency health, CI/CD, and doc hygiene. The repo cannot be called release-ready today, but six focused fixes (≈2-3 cycles) would close that gap.

Top 10 detail

P0 / Critical1. Dependency environment is broken

Evidence. pip check: arxiv 2.4.1 needs requests~=2.32.0 but has 2.33.0; crawl4ai 0.8.6 needs lxml~=5.3 but has 6.1.0; instructor 1.15.1 needs rich<15.0.0 but has 15.0.0. pip-audit: diskcache 5.6.3 (CVE-2025-69872, no fix), litellm 1.83.0 (GHSA-xqmj-j6mv-4862, fix 1.83.7 blocked by transitive click<8.2), pip 26.0.1 (CVE-2026-3219, no fix), ragas 0.4.3 (CVE-2026-6587, no fix).

Fix. Split deps into extras: hybrid, augment, formats, cli-backends, eval, dev. Remove litellm from default path or wait for its click pin to relax. Accept the other three as documented narrow-role risks in a SECURITY.md.

Effort. 0.5 cycle.

P0High2. No CI/CD

Evidence. ls .github fails; the only .github directories live inside vendored skills (.claude/skills/gstack/.github, .tools/ralph-claude-code/.github).

Impact. The 17-step cycle's R1/R2 review gates are all model-driven and local; nothing stops a regression from landing on main.

Fix. .github/workflows/ci.yml with Python 3.12, steps: ruff check, pytest --collect-only, full pytest, pip check, pip-audit, python -m build.

Effort. 0.25 cycle.

P0High3. Package readme points at agent instructions

Evidence. pyproject.toml:6 → readme = "CLAUDE.md". CLAUDE.md is 343 lines of internal runtime policy, not user onboarding.

Fix. One-line change to readme = "README.md". Add test via python -m build && twine check dist/*.

Effort. 5 minutes + CI hook.

P0High4. Partial-write paths leak absolute paths

Evidence. src/kb/mcp/core.py:762:

f"Error[partial]: write to {_rel(file_path)} failed ({write_err}); "

The log-side (line 759) passes write_err to _sanitize_error_str, but the return string interpolates it raw. Same pattern at core.py:881 for kb_save_source. On Windows, OSError.__str__() typically reads "[WinError 5] Access is denied: 'D:\\Projects\\...\\raw\\articles\\foo.md'". Cycle-32 AC3 widened _is_mcp_error_response to include "Error[", so this string is now routed to operator stderr under write-failure conditions. Cycle 32 Threat Model T11 and BACKLOG.md:146 already track it.

Fix. Wrap write_err in _sanitize_error_str(write_err, file_path) at both sites + regression tests that assert no substring matches str(Path.cwd().resolve()) under a simulated PermissionError.

Effort. 30 minutes.

P0High5. "No vectors" conflicts with hybrid search

Evidence. README.md:5 "No vectors. No chunking." · README.md:155 "deletes the vector DB" · README.md:384 v0.10 ships "hybrid search with RRF fusion (BM25 + vector via model2vec + sqlite-vec)" · src/kb/query/engine.py:1 docstring: "BM25 + vector hybrid search + LLM synthesis" · src/kb/query/hybrid.py:1.

Fix. Replace the tagline with "Markdown-first wiki with optional hybrid retrieval". Add an opt-out guide documenting how to disable vector search.

Effort. 2 hours.

P0High6. Test-count drift

Evidence. Live pytest --collect-only -q: 2901. CLAUDE.md + CHANGELOG.md: 2901 (consistent). README.md:9 badge: tests-2850. README narrative cites historical counts 1033, 1177, 2850.

Fix. Generate the badge in CI from pytest --collect-only -q | tail -1; stop citing exact counts in prose.

Effort. 15 minutes + CI hook.

P0High7. PDF support is advertised but absent

Evidence. README.md:142 claims PDF support; config.py:117 includes .pdf; ingest/pipeline.py:1230 reads as UTF-8; pipeline.py:1233 returns "Binary file cannot be ingested" on failed decode. Net behavior: drag a real PDF in → error.

Fix. Remove .pdf from SUPPORTED_SOURCE_EXTENSIONS and update README (20 min), or implement a real PDF extractor with size/page caps (1 cycle).

Effort. 20 minutes (remove) — recommended.

P0High8. Main ingest path has no shared pre-LLM secret/PII filter

Evidence. capture.py:249-264 defines _SECRET_PATTERNS + decoded-base64 candidates; capture.py:771-779 aborts with CaptureError. ingest/extractors.py:336-368 builds the prompt from raw content with no secret scan before call_llm_json(...). BACKLOG.md:219 lists .llmwikiignore + pre-ingest scanner as missing; BACKLOG.md:293-294 notes every ingest sends full content to the API.

Fix. Extract _scan_for_secrets from capture.py into kb/utils/secrets.py. Call at the top of ingest_source and compile_wiki. Add .llmwikiignore. Default deny with override.

Effort. 1 cycle.

P1High9. Scratch planning files in repo root

Evidence. findings.md (cycle-29 scratch), progress.md, task_plan.md (still says [in_progress] Synthesize T1-T3 verdicts), claude4.6.md (40 KB pre-Opus-4.7 copy of CLAUDE.md). None listed in .gitignore; all persist across merges.

Fix. Move to .data/ or delete. Add names to .gitignore. Delete claude4.6.md or move to docs/archive/.

Effort. 10 minutes.

P1High10. Ingest state fan-out is unrecoverable

Evidence. BACKLOG.md:83-84 describes the 11-stage fan-out (summary → N entities → N concepts → index.md → _sources.md → hashes.json → log.md → contradictions.md → N retroactive wikilinks). BACKLOG.md:99-100 R5 notes systemic absence of locking discipline. A crash between manifest-write and log-append leaves manifest saying "ingested" while log shows nothing.

Fix. Backlog's receipt-file recipe (.data/ingest_locks/<hash>.json written first, deleted last; recovery pass replays partials). Introduce with page_lock(page_path): used consistently across _write_wiki_page, _update_existing_page, append_evidence_trail, inject_wikilinks.

Effort. 1-2 cycles.

3. Architecture Review

3.1 Three-layer content model

raw/           (human-owned, immutable)
  ├── articles, papers, videos, repos, podcasts, books, datasets, conversations, assets
  └── captures/   (sole LLM write into raw/; atomised by kb_capture)
wiki/          (LLM-owned, editable by user)
  ├── entities/, concepts/, comparisons/, summaries/, synthesis/
  ├── index.md, _sources.md, _categories.md, log.md, contradictions.md
research/      (human-owned analysis + meta-research)

Durable artifact is Markdown — not an embedding, not a proprietary DB. The user can always read, version, and fork.

3.2 Five-operations cycle

Ingest ──→ Compile ──→ Query
  │                      │
  └── Evolve ←── Lint ←──┘

Compile is thinner than advertised — compile/compiler.py:compile_wiki is a ~50-line loop over ingest_source. No cross-source reconciliation.

3.4 Strengths

  • Raw→wiki→publish explainable in 90 seconds.
  • Markdown-as-artifact is a strong anchor.
  • CLI + MCP split avoids agent-shell lock-in.
  • Reliability primitives: file_lock, atomic_json_write, _validate_path_under_project_root, StorageError redaction.
  • Observability counters + structured logs at cold-load sites.

3.5 Concerns

  • Directory naming inversion: compile/ depends on ingest/ (BACKLOG.md:80-82).
  • 11-stage ingest has no whole-op recovery.
  • No shared graph-cache policy — lint and query rebuild PageRank independently.
  • MCP tools all sync; FastMCP thread pool pressure.
  • Optional features not package-isolated.
Recommendation. Freeze six contracts and gate them with pytest: source ingestion, wiki page schema, query result, publish artifact, error response, dependency extras. Refactor modules behind the contracts, not before.

4. Implementation Review

4.1 Exemplary patterns to preserve

  • SSRF hardening. lint/fetcher.py — SafeTransport, allowlists, private/reserved IP rejection, redirect revalidation.
  • Subprocess safety. utils/cli_backend.py — no shell=True, stdin prompt delivery, argv token rejection, output redaction.
  • Pre-LLM secret scan (capture only). capture.py:249-264, 771-779. Pattern is right; it just isn't shared.
  • Centralised redaction. utils/sanitize.py; used by mcp/core.py:581, 616, 785, 842.
  • Dual-anchor path validation. _validate_path_under_project_root(path, field_name).
  • Atomic writes. temp-file + os.replace.
  • Fair-queue lock stagger (cycle 32). _LOCK_WAITERS counter + position-based stagger clamped to LOCK_POLL_INTERVAL; slot_taken guard survives KeyboardInterrupt.
  • Exception taxonomy. KBError + 5 specialisations; StorageError.__str__ redacts paths.
  • Observability counters. get_dim_mismatch_count, get_bm25_build_count, get_vector_model_cold_load_count, get_sqlite_vec_load_count.
  • Extraction schema cache. LRU via _build_schema_cached(source_type).

4.2 Weaknesses

  • Packaging vs runtime — 6 declared deps, 6+ extras imported under src/kb.
  • 10 modules > 24 KB; config.py god-module.
  • ~50 of 253 test files are test_v0NNN_taskNN.py-style — coverage is hard to audit.
  • No golden/snapshot tests (grep "snapshot\|golden\|syrupy" → 0).
  • Doc-code drift (vectors, PDF, test counts, API-key optionality).
  • Scratch files in repo root (Finding 9).

4.3 Function-local import pattern

Cycles 23-32 introduced function-local imports at CLI→MCP boundaries to preserve boot-lean contracts. Subprocess-probe tests in cycle 31 pin the contract. This is disciplined but subtle — patching must target the owner module (kb.mcp.browse.kb_search), not kb.cli.kb_search. CLAUDE.md documents this; new contributors will get it wrong on the first try.

Recommendation. Promote the cycle-31 subprocess probe into a permanent tests/test_boot_lean.py guard.

  • Versioned test files make coverage hard to navigate.
  • No coverage gate.
  • No mutation testing.
  • No package-extras contract tests.

5.4 Recommendation order

  1. Boot-smoke — pip install . in clean venv; kb --version; no ImportError.
  2. Golden tests with syrupy: frontmatter render, evidence trail, Mermaid, llms.txt, graph.jsonld, siblings, query formats.
  3. Fixture leak guard — autouse monkeypatch of all WIKI_*/RAW_* constants; opt-in --use-real-paths.
  4. Coverage gate in CI; per-module % reporting.
  5. Freeze-and-fold rule after each shipped version.

6. CI/CD Review

Current state: none. Confirmed via ls .github (does not exist). All gates are local + agent-run: R1 Opus / R2 Codex / R3 Sonnet, manual pytest + ruff, manual pip-audit per Step 2 baseline + Step 11 PR-CVE diff + Step 11.5 patch + Step 15 late-arrival warn.

Recommended minimum pipeline

# .github/workflows/ci.yml
name: ci
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }
      - run: pip install -e '.[dev]'
      - run: ruff check .
      - run: pytest --collect-only -q
      - run: pytest -q
      - run: pip check
      - run: pip install pip-audit && pip-audit --strict
      - run: python -m build && pip install twine && twine check dist/*
  extras-smoke:
    strategy: { matrix: { extra: [hybrid, augment, formats, cli-backends] } }
  windows-smoke:
    runs-on: windows-latest

Gate main on test. Run extras-smoke and windows-smoke nightly. Effort: 0.5 cycle minimal; 1 cycle full matrix.

Release automation

Today CHANGELOG.md is hand-maintained with a commit-count convention. A CI step could verify <K> equals git log --oneline main..HEAD | wc -l + 1 before merge. Cycles 30 and 32 both needed commit-count backfills — automating this would prevent recurrence.

Structurally different answer to the same problem. The bet: structure wins long-term over vectors for personal/team knowledge.

8.2 vs Karpathy's gist

The gist is a manual pattern ("ask Claude to write pages"). This is a compiled automation with retroactive linking, trust scoring, contradictions, evolve loop, and Tier-1 publish outputs. The demo/ folder ships a worked example from Karpathy's own X post.

8.3 vs vanilla Obsidian vault

Obsidian is a UI over Markdown with a graph view — it does not ingest, compile, or synthesize. This writes to the same vault format, so a user can adopt Obsidian for free after kb compile. "Meet users where they are" is genuinely strong.

8.4 vs commercial KB tools (Notion AI, Mem, Reflect, Tana)

  • Local-first — all data on disk, git-able, offline-capable.
  • User-owned formats — Markdown + YAML; no lock-in.
  • LLM-agnostic — Claude, Ollama, Gemini CLI, OpenCode, Codex via KB_LLM_BACKEND.
  • Agent-native — 28 MCP tools in Claude Code.
  • Lower polish — no mobile, shared workspace, real-time collab, or SSO. Developer/researcher tool, not enterprise.

8.5 Where differentiation is currently weakened

  • "No vectors" claim (Finding 5) undermines the pitch.
  • PDF support advertised but absent (Finding 7).
  • Heavy install (crawl4ai, playwright, ragas, litellm) belies the "simple Markdown wiki" story.
  • 32 cycles + 251 decision docs in 20 days imply high churn.

9. Innovation

Genuinely novel contributions beyond the Karpathy gist:

  1. Retroactive wikilink injection. New source mentions entity X → existing pages auto-link to X without re-ingest. compile/linker.py:inject_wikilinks_batch — per-page locks, chunked 200-title batches, 500-char cap.
  2. Incremental compile with crash safety. SHA-256 hashes + three-valued manifest (real_hash / failed:{pre_hash} / in_progress:{pre_hash}) + crash-resume audit.
  3. Trust = Bayesian + feedback. "Wrong" penalised 2× vs "incomplete"; per-page posterior updated by kb_query_feedback.
  4. Two-phase refine with audit trail. review/refiner.py — pending row → body mutation → applied/failed flip under file_lock(history_path); attempt-id + sweep-by-attempt-id hardening.
  5. Actor-Critic wiki review. .claude/agents/wiki-reviewer.md + kb_review_page with structured 6-item checklist.
  6. Boot-lean contract. Function-local imports at CLI→MCP boundaries, enforced by a subprocess probe (cycle 31).
  7. Evidence Trail sentinel pattern. Append-only reverse-chronological section inside each page with a machine-maintained sentinel.
  8. Publish-ready outputs. compile.publish ships llms.txt, llms-full.txt, graph.jsonld, sitemap, and per-page siblings from one command.
  9. Output adapters outside wiki/. PROJECT_ROOT/outputs/ is gitignored and separate from the search index — small decision, prevents adapter output from poisoning future queries.
  10. Fair-queue lock stagger (cycle 32). _LOCK_WAITERS + position-based first-sleep stagger clamped to LOCK_POLL_INTERVAL; slot_taken guard is KeyboardInterrupt-safe.

10. Advantages

  • Markdown-first. Durable artifact is human-readable, git-able, forkable.
  • Obsidian-native. Free graph view, backlinks, hover preview.
  • CLI + MCP parity — scripting and agent workflows share a core.
  • 28 MCP tools in Claude Code; "no API key needed" for the main flow.
  • Local-first. Works offline with KB_LLM_BACKEND=ollama / Gemini CLI / OpenCode / Codex.
  • Incremental + crash-safe compile.
  • Trust + contradictions + evolve is meaningfully more than RAG-as-a-library.
  • Strong security primitives where they exist.

12. Risks

  • Trust risk. LLM extraction can hallucinate, omit nuance, overstate certainty.
  • Privacy risk. Main ingest path sends full content to API without pre-ingest secret/PII scan (Finding 8).
  • Dependency risk. 4 open CVEs; broad optional surface.
  • State-consistency risk. 11-stage ingest without whole-op recovery (Finding 10).
  • Scaling risk. Markdown filesystem + graph rebuilds will slow at 10k+ pages. No measured benchmark.
  • Onboarding risk. Mental-model stack is high.
  • Maintenance risk. Cycle-driven development has left stale scratch, drifting test counts, README contradictions.
  • Product-claim risk. "No vectors" and "PDF supported" are load-bearing but unhonored.
  • Supply-chain risk. crawl4ai + playwright against user-supplied URLs in augment execute=True.
  • Lock-contention risk. File locks are not re-entrant; documented order (page → history) can be violated by a new contributor.
  • Vendor-lock subtlety. "No API key needed" is true in Claude Code, false in CLI and output adapters.

13. Ways of Working

13.1 17-step cycle workflow

  1. Requirements gathering
  2. Threat modeling
  3. Brainstorming (optional)
  4. Design evaluation (R1 Opus + R2 Codex in parallel)
  5. Design decision gate (R1/R2 synthesis)
  6. Context7 doc verification (optional)
  7. Implementation plan
  8. Plan gate (Codex coverage matrices)
  9. TDD implementation
  10. CI hard gate (full pytest)
  11. Security verification (threat-model grep conditions)
  12. Existing-CVE patch (optional, Step 11.5)
  13. Documentation update
  14. PR creation
  15. Multi-round PR review (R1 + R2 + R3 if ≥25 items)
  16. Merge + cleanup
  17. Self-review with mandatory skill patching

13.2 Evidence of discipline

  • docs/superpowers/decisions/ contains 251 decision docs across 20 days.
  • 788 total commits on main; 373 in last 7 days ≈ 53/day recently.
  • 32 cycles shipped since 2026-04-05 → 1.6 cycles/day.
  • ~4 artifacts per cycle: design, plan, PR review, self-review.
  • 18 codified feedback rules in memory/feedback_*.md.
  • Mandatory Step 16 skill patches — bugs become process improvements.

13.3 Strengths

  • Explicit gates at design + plan + CI + security.
  • Multi-model adversarial review (Opus + Codex + Sonnet).
  • Self-modifying process via skill patches.
  • Audit trail IS the product.
  • Explicit lesson codification (batch by file, test behavior not signature, ruff-format ordering, etc.).
  • Zero human-in-the-loop bottleneck; cycles don't stall.

13.4 Risks

  • Cycle overload. 1.6 cycles/day; history files hit 277 KB.
  • Decision-doc sprawl. 251 docs in one folder — lookup is O(n).
  • Process is the product. CLAUDE.md crowds out the PRD.
  • Agent reviewers hang. Cycle 32: R2 Codex past 12 min, R1 Opus past 10 min.
  • Scratch leaks. findings.md/progress.md/task_plan.md never cleaned.
  • No observable SLA on wall-clock per cycle.

13.5 Recommendations

  1. Cap cycle velocity — ≤0.5 cycle/day lets docs, tests, feedback catch up.
  2. Promote the retrospective — ≤200-word executive summary at the top of each CHANGELOG entry.
  3. Subdivide decisions/ by cycle range.
  4. Separate PRD from CLAUDE.md.
  5. Add scripts/check_stale_scratch.py as a commit hook.
  6. Fold versioned tests after each cycle.
  7. Track R1/R2 hang rate; switch fallback to deterministic grep harness if >20%.

15. Bottom Line

This repo is stronger than it looks from the outside. The product idea is original and defensible, the implementation has serious reliability and security primitives, and the test volume is real. The bottleneck is not capability — it is that the product contract, dependency story, CI, and doc hygiene have not kept pace with the code. The velocity that produced 32 cycles in 20 days is the same velocity that left scratch files in the repo root, "No vectors" in the README, and readme = "CLAUDE.md" in pyproject.toml.

Three cycles of focused release-hygiene work — most of which is documentation, packaging, CI bootstrap, and a two-line security fix — would produce a repo that matches its code quality with its product contract. At that point, "local-first Markdown knowledge compiler with MCP-native maintenance" becomes a credible claim, not an aspirational one.

If the author continues to treat process as the product, the compounding quality loop (skill patches, codified feedback, decision archive, multi-model review) will keep paying dividends. The thing to protect against is cycle overload — high-frequency iteration only works if the repo surfaces stay clean enough for the next cycle to start from a known state.
