## Testing Policy

- **Per-bug regression test MANDATORY** for all 16 items.
- Tests live in **existing module-scoped test files** (e.g., `tests/test_utils_text.py` for item 11, `tests/test_ingest.py` for items 1/6/14/18, `tests/test_refiner.py` for items 10/13, etc.).
- **NO new `test_phase4_5_theme{N}_*.py` files.**
- Each regression test opens with docstring: `"""Regression: Phase 4.5 CRITICAL item <N> (<short-symptom>)."""` — makes `pytest -k "Phase 4.5"` and grep surface the full story.
- Tests use `tmp_wiki` / `tmp_project` fixtures with `wiki_dir=` threaded through. No writes to production `wiki/` or `raw/`.
- Items 1, 2, 3 fix EXISTING broken tests (no new tests added). Items 7 and 8 get NEW regression tests.

## Error Handling

- No new error types.
- Theme 4 items use `raise ... from e` for cause preservation and `last_error = e` in retry loops.
- No relaxation of existing validation; only tightening.
