## Task Breakdown (6 serial tasks)

### T1: Frontmatter Foundation (D3, D1, D2, P3, P4, L7)

**D3 — Fix `FRONTMATTER_RE`:** The non-greedy `.*?` stops at the first interior `---`.
Fix: use `python-frontmatter` library (already a dependency) as the canonical parser
everywhere, OR tighten the regex to require the closing fence at line start:
`r"\A(---[ \t]*\r?\n(?:(?!---[ \t]*\r?\n).*\r?\n)*---[ \t]*\r?\n?)(.*)"` with DOTALL.
Recommended: tighten regex since `python-frontmatter` has its own edge cases and would
change the parsing contract. The new regex uses a negative lookahead `(?!---[ \t]*\r?\n)`
to skip lines that look like closing fences, only stopping at the actual closing fence.

**D1 — Fix frontmatter guard in refiner:** Current `re.match(r'---\n.*?\n?---', body, re.DOTALL)`
rejects any body starting with `---`. Fix: require at least one `key: value` line between
fences: `r"---\n\s*\w+\s*:.*?\n---"` with DOTALL.

**D2 — BOM handling:** Add `text = text.lstrip("\ufeff")` before frontmatter regex in
`refine_page`. One line.

**P3 — Consolidate evolve/analyzer.py:** Replace inline `re.sub(r"\A---\r?\n.*?\r?\n---\r?\n?", ...)`
at line 95 with `fm_match = FRONTMATTER_RE.match(raw); content = (fm_match.group(2) if fm_match else raw).lower()`.

**P4 — Consolidate review/refiner.py:** Replace divergent `\A\s*---\r?\n...` regex at line 78
with import and use of shared `FRONTMATTER_RE`.

**L7 — Fix semantic.py group index:** Change `fm_match.group(1)` to `fm_match.group(2)` at
line 189. One-char fix, huge impact on consistency checking quality.

**CI gate after T1** — all existing tests must pass since FRONTMATTER_RE is foundational.

### T2: VALID_SOURCE_TYPES DRY Fix (C1)

Delete `VALID_SOURCE_TYPES` from `extractors.py:18`. Add `from kb.config import VALID_SOURCE_TYPES`

### T3: Lint Correctness (L1, L2, L3, L4, L5, L6)

**L1 — Cap `check_cycles`:** `for cycle in itertools.islice(nx.simple_cycles(graph), 100):`
Add warning when truncated: `if len(cycles) >= 100: issues.append("...aborted after 100...")`.

**L2 — Rewrite `_group_by_term_overlap`:** Replace O(n^2) pairwise comparison + 500-page wall
with inverted postings index: `term_to_pages: dict[str, set[str]]`. Build once, emit pairs
from postings with shared-term Counter. No page-count limit needed.

**L3 — `build_graph` accepts pages:** Add `pages: list[dict] | None = None` param. When
provided, iterate pages instead of `scan_wiki_pages + read_text`. Extract wikilinks from
`content` field. Callers in lint/runner.py thread `shared_pages` through.

**L4 — UTC timestamps in trends.py:** `_parse_timestamp` returns aware datetimes always.
Date-only strings get `datetime.combine(d, time.min, tzinfo=timezone.utc)`. `add_verdict`
writes `datetime.now(timezone.utc).isoformat()`.

**L5 — Parse-failure accounting:** Move `overall[vrd] += 1` AFTER the timestamp parse
succeeds (inside the try block, after `ts = _parse_timestamp(...)`). Parse failures
excluded from both overall and period_buckets.

**L6 — `_render_sources` budget:** Add `MIN_SOURCE_CHARS = 500`. Replace `remaining` calc
with `remaining = max(MIN_SOURCE_CHARS, QUERY_CONTEXT_MAX_CHARS - used - len(header) - 20)`.
When `budget <= 0`, return truncation notice instead of full content.

### T4: Data Integrity (D4, D5, D6)

**D4 — Timestamp-based eviction:** Replace `key=lambda x: x[1]["useful"] + x[1]["wrong"] + x[1]["incomplete"]`
with `key=lambda x: x[1].get("last_touched", "")` (ascending = oldest evicted first).
Add `last_touched` timestamp to `add_feedback_entry`. Existing entries without `last_touched`
sort first (evicted before timestamped entries).

**D5 — Claim truncation warning:** Promote `logger.debug` to `logger.warning`. Add
`truncated_claims` key to return dict: `return {"contradictions": contradictions, "truncated_claims": len(new_claims) - max_claims if len(new_claims) > max_claims else 0}`.
Update callers to handle dict return (currently returns list).

**D6 — Exclude current-ingest pages:** In `ingest_source`, compute
`preexisting_pages = [p for p in all_wiki_pages if p["id"] not in pages_created_set]`
before calling `detect_contradictions`. Pass `preexisting_pages` instead of `all_wiki_pages`.

### T5: Query Correctness (Q1, Q2, Q3, Q4)

**Q1 — Per-addition tier budget:** Change `_try_add` to accept a `tier_budget_remaining`
param. For tier 1: `remaining = CONTEXT_TIER1_BUDGET - tier1_used`. If section length
exceeds remaining, skip. For tier 2: `remaining = effective_max - total`.

**Q2 — RRF metadata merge:** On collision: `scores[pid] = {**scores[pid], **result, "score": scores[pid]["score"] + rrf_score}`.
New result fields overwrite old (except score which accumulates). Document that later
lists take precedence for non-score fields.

**Q3 — Smart quote stripping:** Replace `.strip('"')` with:
```python
_QUOTE_CHARS = '"\'\u201c\u201d\u2018\u2019`'
rewritten = rewritten.strip().strip(_QUOTE_CHARS)
```

**Q4 — Centrality status metadata:** Wrap PageRank and betweenness results:
```python
{"values": [...], "status": "ok"}       # normal
{"values": [], "status": "failed"}      # NetworkXError caught
{"values": [], "status": "degenerate"}  # all-uniform / empty
```

### T6: Performance (P1, P2)

**P1 — Bare-slug index:** Before the wikilink loop in `build_graph`:
`slug_index = {pid.split("/")[-1]: pid for pid in existing_ids}`.
Replace 5-subdir loop with `slug_index.get(target)`. O(1) per bare slug.

**P2 — `content_lower` opt-in:** Add `include_content_lower: bool = True` param to
`load_all_pages`. Default `True` for backward compat. When `False`, omit the
`"content_lower": post.content.lower()` line. Update callers that don't need it:
`kb_list_pages`, `build_graph`, `lint_consistency`, `graph/export`, `evolve/analyzer`.
Update `search_pages` to pass `include_content_lower=True`.

## Risk Mitigation

- **D3 (FRONTMATTER_RE):** Highest-risk change. Run full test suite after T1 before
  proceeding. Add golden-file test with interior `---` to catch regressions.
- **D5 return type change:** `detect_contradictions` changes from `list` to `dict`.
  All callers must be updated. Grep for call sites before committing.
- **P2 (content_lower default):** Default `True` ensures zero breakage. Only explicit
  `False` callers change behavior. Safe rollout.
- **L2 (term overlap rewrite):** Behavioral change at scale. Add test with >500 mock
  pages to verify the wall is gone.
