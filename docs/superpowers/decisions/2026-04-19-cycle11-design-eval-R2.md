## Analysis (step-by-step)

I read the cycle-11 requirements, brainstorm, threat model, and `CLAUDE.md`, then checked the exact source/test regions called out in the prompt. The broad design is directionally sound: it is mostly hardening, test coverage, and a small helper relocation. There is no new dependency surface and no new external feature path. The main risks are not in the production changes themselves; they are in several acceptance tests that look behavioural but would not actually exercise the intended contract.

The most important source findings are: `src/kb/ingest/pipeline.py` still has one unlisted string-callee `.get()` site in `_extract_entity_context` (`val.lower()` after `extraction.get(field)`), `tmp_project` does not provide the same scaffold as the manual ingest test, and `src/kb/compile/compiler.py` still imports `page_id` and `scan_wiki_pages` from `kb.graph.builder` inside `detect_source_drift` despite AC5 listing only five internal callers. I also checked the re-export shim risk: a simple `from kb.utils.pages import page_id, scan_wiki_pages` in `kb.graph.builder` does not drop closures or cached state because the current helpers are stateless, but monkeypatch-based tests need to patch the module that the caller actually imported from after AC5.

The CLI ACs need amendment. `python -m kb.cli <cmd> --help` imports `kb.cli`, but Click does not execute the command callback on help, so it will not run the function-local imports in `ingest`, `compile`, `query`, `lint`, `evolve`, or `mcp`. AC8 also needs care: a subprocess test must set `sys.argv = ["kb.cli", "--version"]` before importing `kb.cli`; calling `cli.cli(["--version"])` after importing the module has already missed the import-time short-circuit. A `PYTHONPATH` directory containing only `kb/config.py` is also brittle because it either does not shadow the regular `kb` package or shadows it so `kb.cli` is missing.

## Edge cases / failure modes found

- E1: AC1 -- `_extract_entity_context` still has an unlisted `.get()` site whose callee expects `str`: `val = extraction.get(field)` followed by `val.lower()`. Today `_pre_validate_extraction` covers those four fields, but AC1 should not leave a later crash path dependent on validator parity. -- Recommended test: call `ingest_source(..., extraction={"description": 123, ...})` and assert the failure is the boundary `ValueError`, then add a focused `_extract_entity_context("x", {"description": 123})` unit test if the helper remains directly callable.
- E2: AC1/AC3 -- requirements conflict with current helper semantics. `_coerce_str_field` currently returns `""` for `None`, returns strings unchanged, and raises `ValueError` for `int`, `list`, and `bool`; AC3 says `int -> str(int)`, `list -> joined comma-str`, and `bool -> str(bool)`. -- Recommended test: first decide whether the contract is fail-fast or permissive coercion; then make AC1 and AC3 assert the same policy.
- E3: AC1 -- if `_coerce_str_field` becomes permissive for lists, applying it to `entities_mentioned` / `concepts_mentioned` before `_process_item_batch` would convert list fields into strings, causing `_process_item_batch` to skip all entity/concept pages. -- Recommended test: malformed scalar values should be cleanly rejected or skipped, but valid list values for entities/concepts must still create pages.
- E4: AC2 -- CLI `kb ingest --type comparison` is currently rejected by Click before `ingest_source` because the option choice is `SOURCE_TYPE_DIRS.keys()`. MCP `kb_ingest` and `kb_ingest_content` also reject `comparison`/`synthesis` as "Unknown source_type" before the new library guard can fire. -- Recommended test: cover direct `ingest_source`, `kb_ingest(..., source_type="comparison")`, and `kb_ingest_content(..., source_type="comparison")`; document that the CLI path rejects at option parsing unless CLI validation is changed.
- E5: AC4/AC5 -- AC5 misses `src/kb/compile/compiler.py`, which has function-local imports from `kb.graph.builder` in `detect_source_drift`. Leaving it behind means internal migration is incomplete. -- Recommended test: `rg "from kb\\.graph\\.builder import .*page_id|from kb\\.graph\\.builder import .*scan_wiki_pages" src/kb` should only allow the back-compat shim itself.
- E6: AC4/AC5 -- deleting private `_page_id` will break existing direct tests in `tests/test_v0915_task01.py` and `tests/test_v0914_phase395.py`. This is acceptable only if those tests are migrated in the same cycle. -- Recommended test: update legacy tests to import `kb.utils.pages.page_id`, or keep `_page_id = page_id` as a deprecated alias if test/API compatibility matters more than deletion.
- E7: AC6 -- current `scan_wiki_pages` only scans `WIKI_SUBDIRS`, so root sentinel files (`index.md`, `_sources.md`, `_categories.md`, `log.md`, `contradictions.md`, `purpose.md`, `hot.md`) are already skipped by construction. The test is useful, but it should also create same-named sentinels inside wiki subdirs if the intended guarantee is "never ingest sentinel names anywhere". -- Recommended test: add both root sentinels and `wiki/concepts/index.md` to make the intended skip scope explicit.
- E8: AC7 -- subprocess `--help` does not execute Click callbacks, so it will not exercise function-local imports. -- Recommended test: use `click.testing.CliRunner` or subprocess invocations with minimal valid arguments plus monkeypatched/stubbed downstream functions, or add a hidden/import-smoke command path that actually calls each callback with safe arguments.
- E9: AC8 -- the proposed `PYTHONPATH` `kb/config.py` stub is not reliable for a regular package, and any in-process import of `kb.cli` after pytest has imported `kb.config` is vacuous. -- Recommended test: fresh subprocess: `python -c "import sys; sys.argv=['kb.cli','--version']; import kb.cli; assert 'kb.config' not in sys.modules"` with `PYTHONPATH` limited to repo `src`.
- E10: AC9-AC11 -- missing `sources` key is not reachable through production `search_pages`, because `load_all_pages` always emits `sources`; empty sources is reachable. Non-ISO `updated` is reachable because `_date_str` stringifies arbitrary YAML scalars. Mtime-equals-page-date is reachable. -- Recommended test: keep all three, but label AC9 as defensive-contract coverage unless testing `sources=[]`.
- E11: AC12 -- `tmp_project` is not exact scaffolding for the current manual block. It creates wiki subdirs, raw subdirs, and a plain `# Wiki Log`, but does not create `wiki/index.md`, `wiki/_sources.md`, or frontmatter-bearing log content. The current assertions read `index.md` and `_sources.md`, so a straight replacement will fail or silently stop testing index/source-map updates. -- Recommended test: either enhance `tmp_project` fixture to create canonical index files, or keep two explicit writes for `index.md` and `_sources.md` in this test while removing the directory boilerplate.
- E12: AC12 -- if `ingest_source` is called with both `wiki_dir=tmp_project/"wiki"` and `raw_dir=tmp_project/"raw"`, the old patches for `RAW_DIR`, `WIKI_DIR`, `WIKI_INDEX`, and `WIKI_SOURCES` are not needed. Removing them is good, but it also removes coverage of module-level override behavior. -- Recommended test: keep a separate small regression for module-level fallback only if that behaviour is still intended; otherwise prefer explicit kwargs.
- E13: AC13 -- loading the manifest before/after cannot distinguish a single write from a double write of the same key/value. It proves final state, not write cardinality. -- Recommended test: keep the counter/instrumentation for write count and add a content assertion only as a secondary final-state check; do not claim the content assertion closes the double-write regression by itself.
- E14: AC4/AC5 -- `importlib.reload(kb.graph.builder)` after a true re-export should rebind aliases to the current `kb.utils.pages` functions, but `importlib.reload(kb.utils.pages)` after importing builder will make identity checks diverge until builder is reloaded too. -- Recommended test: avoid over-pinning cross-reload object identity; pin normal import identity and behavioural equivalence.

## Integration / security concerns

- AC2 -- MCP wrappers currently short-circuit `comparison`/`synthesis` as unknown source types. Mitigation: special-case those two values in `kb_ingest` and `kb_ingest_content` before the generic unknown-type branch, returning a sanitized message that names `kb_create_page` and does not include absolute paths or extraction content.
- AC4/AC5 -- monkeypatch tests against `kb.graph.builder.scan_wiki_pages` will not observe callers that migrate to `from kb.utils.pages import scan_wiki_pages`. Mitigation: update tests to patch the caller module's bound symbol or the canonical `kb.utils.pages` symbol before importing the caller.
- AC7/AC8 -- subprocess tests should not inherit a polluted parent `PYTHONPATH`. Mitigation: build a minimal `env` with `PYTHONPATH=<repo>/src` plus required OS keys (`PATH`, and `SYSTEMROOT` on Windows), and keep `shell=False`.
- AC12 -- explicit `raw_dir=` is required for isolation. Passing only `wiki_dir=` still leaves `make_source_ref`, source-type detection, and raw containment using module-level `RAW_DIR` in some paths. Mitigation: make the cleaned-up test call `ingest_source(source, source_type="article", wiki_dir=wiki, raw_dir=raw)`.
- AC1 -- permissive coercion for extraction body fields may convert attacker-controlled lists/dicts into wiki prose, which broadens the write surface. Mitigation: prefer fail-fast for malformed scalar fields, and do not stringify dicts; if list joining is required, only join list items after per-item string validation and sanitization.

## Performance concerns

- AC4/AC5 -- moving `scan_wiki_pages` into `utils/pages.py` has no inherent performance cost. Keep deterministic sorting in the canonical helper; callers already rely on reproducible scan order.
- AC7 -- subprocess CLI smoke tests can become slow if they invoke real command bodies. If AC7 is amended to execute callbacks, stub heavy paths (`compile_wiki`, `query_wiki`, `run_all_checks`, `generate_evolution_report`, `mcp_main`) rather than running real workflows.
- AC13 -- mtime-based write-count tests are likely flaky on Windows and CI filesystems. Avoid timestamp precision as the primary signal.

## Recommended amendments to requirements

- PLAN-AMENDS-DESIGN: AC7 contradicts the intended design goal because `--help` does not exercise function-local imports. Amend AC7 to execute each callback through a safe stubbed invocation rather than help text.
- PLAN-AMENDS-DESIGN: AC8's stub-`kb/config.py` subprocess design is brittle. Amend it to set `sys.argv` before importing `kb.cli` in a fresh subprocess and assert `kb.config` is absent from `sys.modules`.
- PLAN-AMENDS-DESIGN: AC12 says `tmp_project` provides the scaffolding being removed, but it currently does not create `index.md` or `_sources.md`. Amend AC12 to either enhance the fixture or keep explicit index/source-map setup.
- PLAN-AMENDS-DESIGN: AC13's preferred manifest-content assertion cannot distinguish single-write from double-write of the same manifest key. Amend AC13 to retain write-count instrumentation and add manifest-content inspection only as an additional final-state assertion.
- Amend AC5 to include `src/kb/compile/compiler.py`'s function-local imports in `detect_source_drift`.
- Amend AC3 to match the chosen `_coerce_str_field` policy. Current code is fail-fast for non-string values; the written AC3 describes permissive coercion.
- Amend AC2 to specify whether the MCP wrappers and CLI should surface the `kb_create_page` guidance, not only the library-level `ingest_source` call.
