## Post-PR 2-round adversarial review addendum (2026-04-15)

After PR #9 was opened with all 4 themed commits + docs, the `/codex` CLI hit its OpenAI usage quota (reset 2026-04-22), blocking the default post-PR second-opinion step. In lieu of Codex, dispatched a 2-round adversarial review using the same pattern as the feature-dev 3-round design eval:

- **Round 1 (Opus):** architecture, assumptions, scope risk, triage-verification of the two prior Codex majors.
- **Round 2 (Sonnet):** edge cases, concurrency, integration, security regressions.

Both returned `APPROVE-WITH-CONDITIONS` with 4 actionable findings (1 blocker + 3 majors) and 5 deferred items. All 4 actionable findings addressed in commit `4688763` before human merge.

### Decisions

| # | Sev | Finding | Resolution |
|---|---|---|---|
| 1 | BLOCKER | `slug.startswith("untitled-")` false-positives on legit names like `Untitled-Reports` | Tightened to `_is_untitled_sentinel()` helper + regex `^untitled-[0-9a-f]{6}$`; 3 guard sites updated; 2 regression tests |
| 2 | MAJOR | `r"\A\n+"` regex in `refine_page` body rewrite doesn't match `\r\n` (Windows) | `r"\A[\r\n]+"` defense-in-depth; regression test with `\r\n\r\n\r\nActual content.\n` input |
| 3 | MAJOR | `refine_page(wiki_dir=tmp)` silently falls back `history_path` to `REVIEW_HISTORY_PATH` global — same test-isolation class as item 7 | Derive `resolved_history_path = wiki_dir.parent / ".data" / "review_history.json"` when `wiki_dir` present and `history_path=None`; regression test confirms prod mtime unchanged |
| 4 | MAJOR | CJK/emoji titles produce `wiki/summaries/untitled-<hash>.md` — loses discoverability | Fall back to `slugify(source_path.stem)` when title yields a sentinel; regression test: `"😀😀😀"` on `readable-stem.md` → `readable-stem.md` summary |

- The 2-round Opus+Sonnet review found 4 issues the prior automated pipeline (including the branch-level Codex) did NOT flag:
