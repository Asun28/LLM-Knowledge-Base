### Amendment 1 — Reconcile task numbering

The plan has **9 documented TASKs** (TASK 1 through TASK 9 in the per-task subsections). The "Revised order (TDD per skill Step 9)" table re-groups these into **8 commit slots** by pairing impl+test commits. The 9 vs 8 discrepancy is intentional: TASK 6/7/8 are test-only artifacts already paired with their respective code TASKs into the 8 commit slots. The numbering is a presentation choice — there is no missing or duplicate work.

### Amendment 2 — "No multi-file tasks" qualifier

The earlier statement "No multi-file tasks (cycle-4 plan-gate REJECT precedent)" was meant to forbid bundling unrelated files into one commit. It does NOT forbid pairing a test file with its single corresponding implementation file in one commit — that pairing is the natural TDD unit and matches feature-dev Step 9's "implement task N — verify commit — Codex review" loop. Updated wording: "No multi-file tasks across unrelated source files. Test files paired with their corresponding implementation file in the same commit (TDD-cluster) are explicitly allowed."

### Amendment 3 — Explicit ruff format step

Ruff format is run AFTER all Edits per pass per user-memory `feedback_ruff_edit_ordering`. Concrete sequence per commit:
1. Edit / Write the source file(s).
2. Edit / Write the test file in the same commit.
3. Run `ruff format src/ tests/` ONCE (after both edits land).
4. Run `ruff check src/ tests/ --fix` ONCE (let ruff fix import sort + trivial style).
5. Re-Read both files (mandatory if any subsequent Edit needed).
6. Run `python -m pytest tests/test_cycle13_*.py -x` to verify the new tests pass.
7. `git add` + `git commit`.

This sequence appears verbatim at the top of each Step-9 task execution.
