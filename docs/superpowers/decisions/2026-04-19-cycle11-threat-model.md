## Analysis

Cycle 11 is an unusually narrow hardening cycle: no new dependencies, no new MCP tools, no user-facing feature surface. The bulk of the 14 ACs are test-only (AC3, AC6, AC7, AC8, AC9, AC10, AC11, AC12, AC13) or additive defensive guards (AC1 `_coerce_str_field` extension, AC2 `comparison`/`synthesis` reject-early, AC4/AC5 re-export shim). The real security-relevant moves are (a) tightening the ingest-extraction type-coercion perimeter to 6+ more read sites in `pipeline.py`, (b) closing a silent dead path for `source_type in {"comparison","synthesis"}` that currently emits a near-empty summary rather than raising, (c) relocating `page_id` + `scan_wiki_pages` from `kb.graph.builder` to `kb.utils.pages` with a back-compat re-export, and (d) adding subprocess smoke tests that spawn a fresh Python interpreter with controlled `PYTHONPATH`. Because each of these is additive-over-existing-controls (the pre-validate pass already runs `_coerce_str_field` on 8 summary-string fields at `pipeline.py:84-85`; `_validate_wiki_dir` already exists; the CLI already short-circuits `--version` at `cli.py:15-19` before importing `kb.config`), the blast radius for a regression is bounded — but consistency-drift risks are non-trivial and directly in-scope for Step 11.

Two concerns dominate. First, **import-path consistency drift from the AC4/AC5 re-export shim**: five internal modules switch from `from kb.graph.builder import page_id, scan_wiki_pages` → `from kb.utils.pages import ...` in the same commit that converts `kb.graph.builder.page_id` into a re-export. Today `kb.utils.pages._page_id` (private, line 29-31) and `kb.graph.builder.page_id` (public, line 27-34) are **already divergent**: the private helper uses `str(...).replace("\\", "/").removesuffix(".md").lower()` while the public uses `page_path.relative_to(wiki_dir).as_posix().removesuffix(".md").lower()`. Both produce identical output for in-tree paths but differ on edge cases (symlinks, resolved-vs-unresolved paths, non-child paths). The cycle-11 design calls for deleting the private duplicate and making the public one canonical — so the semantic drift is fixed — but any test that still monkeypatches `kb.graph.builder.page_id` will now patch a re-export binding that may be shadowed by module-local imports done before the patch. Existing `test_backlog_by_file_cycle7.py` already patches `builder.scan_wiki_pages` (line 431) and `linker.scan_wiki_pages` (line 390); after cycle 11, the patched `graph.builder` symbol is a re-export alias, and callers that `from kb.utils.pages import scan_wiki_pages` at top of module will NOT see the patch. This is a latent test-integrity bug that Step 11 must grep for. Second, **the AC2 dead-path closure adds a new error-return surface**: `ingest_source(..., source_type="comparison")` currently takes a silent path through `_build_summary_content` emitting an empty shell summary. AC2 will raise `ValueError` (or return `{"error": ...}`) instead. If the error message interpolates the attacker-controlled `source_type` string verbatim into an error response that reaches the MCP client, and if that client is a downstream LLM that treats errors as instructions, we reopen the cycle-10 T1 concern. The cycle-10 `_sanitize_error_str` redactor should already cover this via the `kb_ingest` wrapper, but Step 11 must verify that AC2's error text does not include absolute paths or user-controlled file contents beyond the `source_type` token itself.

A third, lower-tier concern is **subprocess test-fixture pollution (AC7/AC8)**: the requirements spec calls for `env={**os.environ, "PYTHONPATH": str(repo / "src")}`. A developer or CI runner with a pre-existing `PYTHONPATH` that prepends a hostile path would not be sanitized by this pattern — the test would still import the hostile module if it shadows a `kb.*` symbol. The fix is to pass `env={"PYTHONPATH": str(repo / "src"), "PATH": os.environ.get("PATH","")}` rather than spreading `os.environ`. Step 11 should flag this as a "tighten test isolation" item but not block the cycle on it (single-user local-only project).
