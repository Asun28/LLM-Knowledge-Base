# Cycle 44 Design Eval R2 (Codex — edge cases / integration / test invariants)

## Analysis (≥2 paragraphs explicit reasoning)
The highest-risk issue is the AC25 signature reversal. Current grep shows `src/kb/utils/io.py:144:def atomic_text_write(content: str, path: Path) -> None:` and 24 direct source call sites still pass `(content, path)` (`src/kb/capture.py:472`, `src/kb/compile/publish.py:205,286,366,502,519,560`, `src/kb/compile/linker.py:296,549`, `src/kb/ingest/evidence.py:244`, `src/kb/ingest/pipeline.py:219,405,755,804,815,869`, `src/kb/lint/augment.py:526,1026`, `src/kb/lint/checks.py:141`, `src/kb/mcp/core.py:920`, `src/kb/query/formats/__init__.py:68,69,72`, `src/kb/review/refiner.py:271`, `src/kb/utils/pages.py:252`). There are also 10 direct test call sites (`tests/test_backlog_by_file_cycle2.py:267`, `tests/test_v01008_ingest_pipeline_fixes.py:217`, `tests/test_v0914_phase395.py:408,425`, `tests/test_v0915_task11.py:27,35,42,50,58`, `tests/test_v0916_task02.py:94`). Changing the positional order to `(path, content)` would silently swap arguments across the codebase unless every caller is migrated in the same change; the safer design is to preserve `(content, path, *, exclusive=False)`.

The second systemic risk is patch-site drift after module splits. Re-export shims preserve importability, but they do not make `patch()` and `monkeypatch.setattr()` transparent when the moved function reads globals from its new module. This affects `kb.lint.checks.WIKI_DIR` / `RAW_DIR` in `tests/test_cli.py:88-89`, many `kb.lint.augment.*` and `kb.lint._augment_*` sites, and `kb.mcp.core.*` sites that patch constants or tool functions before invoking re-exported functions. The requirements cite cycle-19 L1 and C42-L3, but AC14/AC15 name only two files while grep finds additional string-form and reference-form sites. M3 has the same problem: after moving `kb_ingest`, `kb_ingest_content`, `kb_save_source`, `kb_compile_scan`, and `kb_capture` out of `core.py`, tests patching `kb.mcp.core.PROJECT_ROOT`, `RAW_DIR`, `SOURCE_TYPE_DIRS`, `ingest_pipeline`, or `core_mod.kb_ingest_content` need either canonical target migration or a deliberate compatibility proxy.

The test-fold work is cleaner but not complete as written. `rg -n "sanitize|_sanitize|conversation_context|context" tests/test_cycle12_sanitize_context.py tests/test_mcp_core.py` found all sanitizer hits in `tests/test_cycle12_sanitize_context.py` and none in `tests/test_mcp_core.py`; there is no duplicate to drop before the fold. However the source file currently has two test functions, one parametrized over two branches (`tests/test_cycle12_sanitize_context.py:35` and `:105`), so the fold contributes three collected tests, not the `>=5` required by AC21. AC21 also names empty-list, list-over-limit truncation, and content-passthrough coverage that is not present in the current folded source; those tests must be added, not just moved.

## Failure modes / integration concerns
- Test fold redundancy risk (cycle-17 L3): `tests/test_mcp_core.py` has no current sanitizer hits, while `tests/test_cycle12_sanitize_context.py:35` and `:105` contain the sanitizer tests. No duplicate fold risk was found, but AC21 must add direct sanitizer cases because the existing file contributes only three collected cases.
- Behavioral test self-check (cycle-16 L2): for AC29, first implement the cache-key behavior, then temporarily mutate production back to `(path, mtime_ns)` only and verify `test_load_page_frontmatter_mtime_collision` fails. For AC30, temporarily mutate the stale-lock `unlink` path in `file_lock` to a no-op and verify the new stale-lock test fails.
- Tool registration (cycle-23 L5): current `src/kb/mcp/__init__.py:37` imports `browse, core, health, quality` in `_register_tools()`. After M3, that PEP 562 `__getattr__` path must import `ingest` and `compile` too, or their `@mcp.tool()` decorators will not fire on `from kb.mcp import mcp`; threat model T4 requires all 29 tools (`threat-model.md:31`).
- Re-export brittleness (C42-L5): every `from .X import Y` line in new package `__init__.py` files and in `core.py` compatibility shims needs `# noqa: F401  # re-exported for backward compat (cycle-23 L5)` plus a one-line nearby comment explaining that ruff must not strip compatibility exports.
- Private augment modules: AC11 removal of `_augment_manifest.py` and `_augment_rate.py` is incompatible with current tests unless compatibility shims remain or every listed test is migrated. This is not limited to AC14/AC15.
- M3 constant patches: functions moved to `ingest.py` or `compile.py` will read their new module globals. Re-exporting functions from `core.py` does not make `monkeypatch.setattr(core, "RAW_DIR", ...)` affect those globals.
