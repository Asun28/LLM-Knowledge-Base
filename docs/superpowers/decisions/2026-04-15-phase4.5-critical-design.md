## Decisions

### Q1 — Commit granularity: `4 theme commits` (HIGH)
Matches 4 root-cause themes. Per-theme commits tell coherent story; 16 per-item commits over-fragment; squash destroys attribution. Senior-engineer test: theme commits are minimum-viable unit.

### Q2 — Theme 4 merger: `Keep Theme 4 separate` (HIGH)
Error-chain preservation is a distinct semantic invariant from contract-consistency. Merging saves 1 commit at the cost of conceptual blur. No merge conflict (different functions in `utils/llm.py`).

### Q3 — Theme 3 subagent task split: `ONE subagent task for Theme 3 main` (MEDIUM)
5 items at same abstraction level. `pipeline.py` already touched by Theme 2, so splitting doesn't reduce coupling. refiner.py item 13 goes in refiner-bundle (Q9-new), leaving Theme 3 main with 4 items: 11, 12, 14, 15.

### Q4 — Regression-test file convention: `Append to existing module-scoped test files` (HIGH) — OVERRIDES prior design's `test_phase4_5_theme*.py` scheme
Per CLAUDE.md Testing convention: module-scoped tests, not phase-scoped. Each regression test has docstring `"""Regression: Phase 4.5 CRITICAL item N (<short-symptom>)."""` so `pytest -k "Phase 4.5"` or grep still surfaces the theme story. Theme-scoped files become archaeology; drive-by maintenance cost.

### Q5 — TODO comment placement: `At the race-prone _is_duplicate_content site` (HIGH) — OVERRIDES scope-gate ambiguity
Verbatim text: `# TODO(phase-4.5-high): wrap load_manifest→mutate→save_manifest in file_lock(manifest_path); CRITICAL fix removed the double-write, single-writer RMW race remains`
TODO comments are actionable directives to future maintainers, not epitaphs. Placing at the site where work is needed, not at the site of deletion.

### Q6 — Pre-commit test-coverage hook: `DO NOT add` (HIGH)
Same "preventive infrastructure is a drive-by edit" principle that deferred Theme 5's `verify_docs.py`. User asked for bug fixes with tests + review, not enforcement infrastructure. Open as its own BACKLOG item.

### Q7 — Codex review cadence: `Per-theme Codex review (6 runs matching 6 subagent tasks)` (HIGH)
feature-dev Step 4a specifies per-task Codex. 4 themes + 2 cross-theme bundles = 6 subagent tasks = 6 Codex runs. Focused context beats consolidated noise.

### Q8 — Security review gate: `MANDATORY at Step 4b` with 4-item checklist (HIGH)
PR touches 6 security-sensitive files (ingest pipeline, query engine, text utils, markdown, refiner, io). Checklist:
1. `slugify` CJK fallback collision-safety (item 11)
2. `raw_dir` derivation sandbox-escape via `..` (item 7)
3. `refine_page` `file_lock` path bounds (item 13)
4. `WIKILINK_PATTERN` tightening preserves path-traversal protection (item 12)

### Q9 (surfaced) — refiner.py bundling: `One refiner-bundle subagent task for items 10 + 13` (HIGH)
Items 10 (Theme 2) and 13 (Theme 3) both edit lines 13-133 of `review/refiner.py`. Scope-gate Condition 3 mandates bundling. Bundle output SPLIT across Theme 2 commit (item 10) and Theme 3 commit (item 13).

### Q10 (surfaced) — lint/runner.py bundling: `One lint-runner-bundle subagent task for items 8 + 9` (HIGH)
Items 8 (Theme 1) and 9 (Theme 2) both edit `run_all_checks` at lines 53-98. Scope-gate Condition 2 mandates bundling; item 8 first. Bundle output SPLIT across Theme 1 commit (item 8) and Theme 2 commit (item 9).

### Q11 (surfaced) — pipeline.py multi-theme: `Serial subagent execution` (HIGH)
`ingest/pipeline.py` touched by items 6 (Theme 2), 14 (Theme 3), 18 (Theme 4). Parallel subagents = merge conflicts. Serial with rebase = CLAUDE.md simplicity.
