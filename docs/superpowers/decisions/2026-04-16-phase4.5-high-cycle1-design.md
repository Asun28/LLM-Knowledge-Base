## Decisions

### Original Questions

**Q1 — H5 raw_dir tautology: DROP try/except with doc comment.** CONFIDENCE: HIGH.
- Arguments for drop: path is constructed internally; no untrusted input flows in; "don't add error handling for scenarios that can't happen" (CLAUDE.md).
- Arguments for anchor-PROJECT_ROOT: defense-in-depth. Counter: violates "every changed line traces directly to the request."
- Rationale: Lower blast radius; cycle-1 already constructs path safely.

**Q2 — H10 field sanitization: SEPARATE `sanitize_extraction_field` helper.** CONFIDENCE: HIGH.
- Arguments for separate: `yaml_escape` contract (YAML double-quoted scalars) is distinct from markdown-body sanitization contract.
- Arguments for folded: fewer helpers to remember. Counter: blurs both contracts.
- Rationale: Keep contracts focused.

**Q3 — H12 evidence trail sentinel: HTML COMMENT `<!-- evidence-trail:begin -->`.** CONFIDENCE: HIGH.
- Arguments for sentinel: backward-compat with existing pages; proactive immunization.
- Arguments for refuse-multi-anchor: reactive failure mode. Counter: surfaces only after attack.
- Rationale: Lower-blast-radius + forward-compat. Combined with H10's `^##+ Evidence Trail` strip, doubly protected.

**Q4 — H17 rebuild timing: HYBRID — mtime-check + `_skip_vector_rebuild` batch param.** CONFIDENCE: HIGH.
- Arguments for hybrid: avoids 200× perf cliff in `compile_wiki --full`.
- Arguments for naive always-rebuild: simpler. Counter: measured 200s regression is a release-blocker.
- Rationale: Mtime + opt-out matches "opt-in flag > always-on behavior." See also Q_I.

**Q5 — H7 `append_wiki_log` wiki_dir: REQUIRED parameter.** CONFIDENCE: HIGH.
- Arguments for required: the default IS the bug; 12 sites auditable in one PR.
- Arguments for optional: less churn. Counter: degrades to same bug shape.
- Rationale: "Internal-only > public API" + bug is exactly default-fallback-exists.

**Q6 — Test organization: APPEND to existing module-scoped test files.** CONFIDENCE: HIGH.
- Arguments for append: cycle-1 CRITICAL C7 precedent; avoid versioned-file archaeology.
- Arguments for new files: phase-scoped organization. Counter: versioned test files are archaeology burden.
- Rationale: Carry forward cycle-1 convention.

**Q7 — H17 import-failure policy: MODULE-LOAD-TIME WARN + `_hybrid_available` flag.** CONFIDENCE: HIGH.
- Arguments for WARN-once: single clear signal; observable degradation.
- Arguments for silent-skip: less noise. Counter: silent degradation is the bug.
- Rationale: Match BACKLOG R2 explicit ask for observable signal.

**Q_C — `_update_existing_page` exception fall-through: ADD to Theme 4.** CONFIDENCE: HIGH.

- Arguments for defer: +1 item. Counter: lowest-cost item in cycle.
- Rationale: Essentially 1-line typo fix with concrete user impact.

**Q_D — H17 scope: STAYS in cycle 1, narrowed via Q4/Q_I; DEFER lifecycle items.** CONFIDENCE: MEDIUM.
- Arguments for keep-narrow: hybrid shape contains blast radius; ships the documented "BM25-only lie" fix.

**Q_E — Cycle split: SINGLE PR for all 22 items.** CONFIDENCE: HIGH.
- Arguments for single: cycle-1 precedent (16 items, 1 PR, 4 theme commits); reverts at theme-commit granularity.
- Arguments for split: smaller PRs. Counter: CHANGELOG/BACKLOG churn doubles.
- Rationale: +6 items are small (~250 LOC production); single PR matches cadence.

**Q_F — Multiprocessing tests: DEFER to test-infra cycle.** CONFIDENCE: MEDIUM.
- Arguments for defer: slow + flaky on Windows CI; cycle-1 precedent shipped without mp tests.
- Arguments for add-one: validation gap closes. Counter: gap documented + reproducible.
- Rationale: Lower-flake path matches cycle-1 precedent.

**Q_G — Atomic vec-DB rebuild: ACCEPT partial-rebuild risk + add `_index_cache.clear()`.** CONFIDENCE: MEDIUM.
- Arguments for accept: crash window <1s; recovery on next ingest; cache-clear is cheap immediate fix.
- Arguments for full atomic: bigger refactor. Counter: defer to lifecycle cycle.
- Rationale: Surface-area discipline.

**Q_H — H1 audit-ordering: ACCEPT residual risk; new BACKLOG entry.** CONFIDENCE: MEDIUM.
- Arguments for accept: adopting R2 fix doubles H1 blast radius; separate item.
- Arguments for adopt: close the gap. Counter: refactor needs own design pass.
- Rationale: Don't gold-plate; track follow-up.

**Q_I — H17 `compile_wiki` perf cliff: PRIVATE `_skip_vector_rebuild` param + single tail-rebuild.** CONFIDENCE: HIGH.
- Arguments for hybrid: 200× slowdown is release-blocker; opt-in private param has zero MCP-surface impact.
- Arguments for simple: one control point. Counter: both controls serve different purposes (correctness + perf).
- Rationale: Only shape that meets correctness AND keeps batch perf.

**Q_J — H10 scope: EXPAND to cover `_update_existing_page:280-283` context-append.** CONFIDENCE: HIGH.
- Arguments for expand: same attacker-controlled extraction_json flow; +1 touchpoint same helper.
- Arguments for narrow: scope discipline. Counter: half-closed persistence path is worse than either closed or open.
- Rationale: H10's stated goal fails without the expansion.

**Q_K — Bundle R4 #38 + R4 #43: ADD both to Theme 3.** CONFIDENCE: HIGH.
- Arguments for bundle: 3-5 line fixes each; same attack surface; closes Theme 3 story.
- Arguments for defer: scope creep. Counter: <15 LOC total; cycle-2 leftover burden.
- Rationale: Theme 3's real shape is "input-validation hardening" — these fit exactly.

**Q_L — H14 XML sentinel + system instruction: ADD instruction now to `build_review_checklist()`.** CONFIDENCE: HIGH.
- Arguments for add: XML tags alone don't prevent LLM instruction-following (Anthropic red-team findings).
- Arguments for defer: wiki-reviewer.md agent prompt is separate. Counter: 1-3 LOC in checklist is free correctness win.
- Rationale: Defensive instruction beside the sentinel is cheap and closes the vector.

**Q_M — H12 sentinel placement: END-OF-SECTION (option B).** CONFIDENCE: HIGH.
- Arguments for end-of-section: forward-looking threat model; "reversible > irreversible" (no pre-upgrade migration).
- Arguments for above-existing (A): uniform anchor. Counter: orphans pre-upgrade entries.
- Arguments for migrate (C): destructive refactor.
- Rationale: Pre-upgrade entries are already-shipped historical data; sentinel is for FUTURE inserts.

**Q_N — H16 error-tag format: MODULE-LEVEL CONSTANT `ERROR_TAG_FORMAT` in `mcp/app.py`.** CONFIDENCE: HIGH.
- Arguments for constant: grep-able; documented category list; stable contract for agent consumers.
- Arguments for string-literal convention: less ceremony. Counter: evolves into divergence.
- Rationale: Single source of truth for error-category set. Categories: `prompt_too_long`, `rate_limit`, `corrupt_page`, `invalid_input`, `internal`.
