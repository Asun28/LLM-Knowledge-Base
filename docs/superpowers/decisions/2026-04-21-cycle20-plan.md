**Change (production):**
1. `_write_wiki_page` signature: add `, *, exclusive: bool = False` keyword-only param at end.
2. Body: when `exclusive=True`, replace `atomic_text_write(...)` with the O_EXCL block from AC8 (POSIX `O_NOFOLLOW` guard, write-phase cleanup via unlink on exception). `_write_wiki_page` still calls `append_evidence_trail` internally AFTER the O_EXCL write; `append_evidence_trail` has its OWN internal `file_lock(page_path)` (cycle-1 H1 — verified). No caller-held file_lock wraps this call.
3. On `FileExistsError`, raise `StorageError("summary_collision", kind="summary_collision", path=effective_path) from e`. When `exclusive=False`, continue calling `atomic_text_write` as today.
4. `_update_existing_page` body: wrap the entire read → modify → `atomic_text_write` → `append_evidence_trail` span in `with file_lock(page_path):`. Add `from kb.utils.io import file_lock` if not already imported. **CRITICAL (plan-gate amendment):** because `append_evidence_trail` itself acquires `file_lock(page_path)` internally and `file_lock` is NOT re-entrant, the RMW span inside `_update_existing_page` must acquire file_lock ONCE around the body read + atomic_text_write, and `append_evidence_trail` must be called AFTER file_lock release. Restructure the function: `with file_lock(page_path): ... read → modify → atomic_text_write; # lock released here ... append_evidence_trail(...)` AFTER the `with` block.
5. At `pipeline.py:1254` summary caller: **NO file_lock wrapper (plan-gate amendment).** Just `try: _write_wiki_page(summary_path, ..., exclusive=True) except StorageError as err: if err.kind == "summary_collision": _update_existing_page(summary_path, source_ref, verb="Summarized") else: raise`. Serialisation happens via (a) `append_evidence_trail`'s own internal `file_lock(page_path)` during A's evidence write and (b) `_update_existing_page`'s new unconditional `file_lock(page_path)` during B's merge. Both acquire the same `.lock` sidecar; B blocks until A completes. No self-deadlock.
6. No caller-held file_lock.
7. AC5 first half: locate `_run_ingest_body`'s outer `except Exception as e:` (grep: `def _run_ingest_body` in `pipeline.py`). Narrow to `except (IngestError, OSError, ValueError) as e:` and add `except Exception as exc: raise IngestError(str(exc)) from exc` catch-all.
8. AC7 regression test dispatches through real `ingest_source` with injected `OSError`.

- **AC5 outer-except narrowing could propagate previously-swallowed exceptions.** Mitigation: AC5 is scoped to 2 sites where `except Exception` wraps-and-continues (IngestError + QueryError taxonomy). Tests AC7 + AC7-mirror prove the wiring.

- **AC14 MCP tool count drift.** R2 noted actual count is 27 (not 26). AC4 CLAUDE.md update must align.
