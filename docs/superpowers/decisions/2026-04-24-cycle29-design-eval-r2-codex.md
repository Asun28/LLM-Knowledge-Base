## Edge-Case Analysis
### 1. Concurrency
`rebuild_indexes` serializes only the manifest unlink through `file_lock(manifest_path, timeout=1.0)`, while vector DB and `.tmp` unlinks are unlocked (`src/kb/compile/compiler.py:670-690`, `src/kb/compile/compiler.py:708-710`). Two concurrent `rebuild_indexes` calls are mostly idempotent, but an `exists()`/`unlink()` race can still produce "cleared" after another process recreates a file (`src/kb/compile/compiler.py:674-676`, `src/kb/compile/compiler.py:688-690`). AC2 validation before unlink is not atomic with the destructive operation; the threat model explicitly places validation before lock/unlink, so Step 5 must decide whether this residual TOCTOU is accepted or narrowed (`docs/superpowers/decisions/2026-04-24-cycle29-threat-model.md:50`).

### 2. Cross-Platform
The existing code documents dual-anchor validation for `wiki_dir`, checking both the caller-supplied absolute path and the `.resolve()` target (`src/kb/compile/compiler.py:638-656`). It also documents that vector `Path.unlink` removes symlinks themselves rather than targets (`src/kb/compile/compiler.py:682-684`). Windows symlink privilege concerns are already recognized by the brainstorm, which recommends monkeypatching `Path.resolve` rather than requiring a real symlink on Windows (`docs/superpowers/decisions/2026-04-24-cycle29-brainstorm.md:80-88`). UNC path and junction-specific behavior is not specified in the codebase: not found.

### 3. Error-Message Consistency
`append_wiki_log` normalizes `|`, `\n`, `\r`, and `\t` before writing the markdown log entry, so embedded newlines in `result["vector"]["error"]` become a single physical log line (`src/kb/utils/wiki_log.py:91-104`). A bytes-as-string representation has no special branch; it is just part of the message passed through the same field sanitizer (`src/kb/utils/wiki_log.py:102-104`). AC1 should still add a regression with an embedded newline in the simulated `OSError` so the deterministic one-line contract is pinned to behavior, not just assumed.

### 4. Integration With Existing Callers
Grep found these direct helper call shapes: `wiki_dir+hash_manifest+vector_db` (`tests/test_cycle23_rebuild_indexes.py:100-104`, `tests/test_cycle23_rebuild_indexes.py:237-240`), `wiki_dir+ghost hash_manifest+ghost vector_db` (`tests/test_cycle23_rebuild_indexes.py:127-130`), `wiki_dir` only (`tests/test_cycle23_rebuild_indexes.py:150`, `tests/test_cycle23_rebuild_indexes.py:173`, `tests/test_cycle23_rebuild_indexes.py:199`, `tests/test_cycle25_rebuild_indexes_tmp.py:48`, `tests/test_cycle25_rebuild_indexes_tmp.py:74`, `tests/test_cycle25_rebuild_indexes_tmp.py:106`), and `wiki_dir+vector_db` (`tests/test_cycle25_rebuild_indexes_tmp.py:141`). Existing override fixtures place paths under the patched temp project root (`tests/test_cycle23_rebuild_indexes.py:90-99`, `tests/test_cycle25_rebuild_indexes_tmp.py:127-141`), so AC2 should not break them if it patches the right module attribute. Cycle-25 `in_progress:` manifest markers are a compile-manifest lifecycle issue; `rebuild_indexes` still unlinks the whole manifest under lock, and AC2 path validation does not alter marker semantics (`CLAUDE.md:142`, `src/kb/compile/compiler.py:670-676`).

### 5. Reload-Leak Hazards
`compiler.py` snapshot-imports `PROJECT_ROOT` from `kb.config`, so the correct test patch target for `rebuild_indexes` validation is `kb.compile.compiler.PROJECT_ROOT`; patching only `kb.config.PROJECT_ROOT` is insufficient after import (`src/kb/compile/compiler.py:9-16`, `src/kb/compile/compiler.py:638`). Existing tests already patch both in many places, and `tmp_kb_env` mirror-rebinds imported `kb.*` globals that still match original config values (`tests/test_cycle23_rebuild_indexes.py:90-93`, `tests/conftest.py:244-255`). The named CLAUDE.md red-flag table was not found; the closest cross-session reload/snapshot warning is the testing note at `CLAUDE.md:195`.

### 6. Path.resolve() vs Path.absolute()
The implementation uses `.resolve()` for `PROJECT_ROOT` and `wiki_dir`, not `.absolute()` (`src/kb/compile/compiler.py:638`, `src/kb/compile/compiler.py:651-656`). The existing comment says relative inputs skip the literal pre-check because `.resolve()` absolutifies against CWD, which may differ from project root (`src/kb/compile/compiler.py:644-646`). Windows junction-following behavior for `Path.resolve()` is not specified in the repo: not found, so the AC2 design should not claim equivalence across POSIX symlinks, Windows symlinks, and junctions without an explicit Step-5 decision or test strategy.

### 7. AC1 Failure-Mode Completeness
The audit message is built before the `try`, then `append_wiki_log` is called inside a `try` that catches only `OSError` and leaves `audit_written=False` on failure (`src/kb/compile/compiler.py:752-761`). `append_wiki_log` itself retries once on `OSError` and then raises to the caller (`src/kb/utils/wiki_log.py:154-158`). Fallback for a renderer exception or a non-`OSError` from `append_wiki_log` is not found; AC1 should keep the renderer pure and side-effect-free, or explicitly decide that current narrow failure handling remains the contract.

### 8. Test Hygiene
Cycle-24 design treats divergent-fail tests as load-bearing, so AC1 tests should assert against the final `rebuild-indexes` log entry, not an unpositioned whole-file substring (`docs/superpowers/decisions/2026-04-23-cycle24-design.md:16`, `src/kb/utils/wiki_log.py:102-104`). `warn:` was not found in existing rebuild-index log lines, and `tmp:` currently appears in the result-error test rather than the audit log, so asserting `vector=cleared (warn: tmp:` on the last log line should fail after a renderer revert (`tests/test_cycle25_rebuild_indexes_tmp.py:113-116`, `src/kb/compile/compiler.py:752-756`). AC2 outside-root tests must use a valid in-root `wiki_dir` plus an outside override and an unlink spy, otherwise the pre-existing `wiki_dir` validator could satisfy the test for the wrong reason (`docs/superpowers/decisions/2026-04-24-cycle29-requirements.md:47-52`). AC4 and AC5 source scans must be section-scoped: scanning all code would collide with the legitimate `_PROMPT_TEMPLATE` cache variable, and scanning all BACKLOG text would collide with the surviving HIGH-Deferred cold-load topic (`src/kb/capture.py:310-318`, `BACKLOG.md:109`).

## Step-5 Gate Questions
1. Should AC2 explicitly accept the residual path-validation TOCTOU, or require a narrower mitigation such as immediate pre-unlink revalidation and rejection of symlink/junction parent chains?
2. Should AC1's compound renderer normalize control characters itself, or is relying on `append_wiki_log`'s existing sanitizer the intended contract?
3. Should the CLI status renderer mirror AC1's compound `cleared (warn: ...)` token, since it currently drops `result["vector"]["error"]` when `cleared=True` (`src/kb/cli.py:550-560`)?
4. For AC2 tests, is `kb.compile.compiler.PROJECT_ROOT` the required patch target, with `kb.config.PROJECT_ROOT` only patched for fixture consistency?
5. Should the symlink-escape test use the brainstorm's `Path.resolve` monkeypatch strategy for portability, or also add an optional real symlink/junction test with Windows skips?
6. Does "None overrides skip validation" remain acceptable if `HASH_MANIFEST` has drifted outside `PROJECT_ROOT`, or should defaults be validated in a separate future cycle?
7. Should AC1 broaden the audit failure catch beyond `OSError`, or preserve the current narrow fallback and document that non-`OSError` failures are programmer errors?
