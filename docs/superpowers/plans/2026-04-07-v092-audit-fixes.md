## Fix Groups

### Group 1: Ingest Pipeline Fixes
**Files:** `src/kb/ingest/pipeline.py`, `src/kb/cli.py`, `src/kb/mcp/app.py`
**Test:** `tests/test_ingest_fixes_v092.py` (8 tests)

- [x] **Fix 1 — Flawed regex in `_update_existing_page()`:** Replaced negative lookahead with `finditer` last-match approach so new sources always insert after the last existing `source:` entry, not mid-list.
- [x] **Fix 2 — Silent exception in `_update_existing_page()`:** Added `logging.warning` with specific exception types (`yaml.YAMLError`, `OSError`, `ValueError`, `AttributeError`) instead of bare `except`.
- [x] **Fix 3 — Surface `pages_skipped` in CLI and MCP:** `_format_ingest_result()` in `kb.mcp.app` now renders a "Pages skipped (N):" section when the ingest result includes `pages_skipped`.

### Group 2: Semantic Lint + Query + MCP Fixes
**Files:** `src/kb/lint/semantic.py`, `src/kb/query/engine.py`, `src/kb/query/bm25.py`, `src/kb/mcp/browse.py`, `src/kb/mcp/health.py`
**Test:** `tests/test_lint_query_fixes_v092.py` (12 tests)

- [x] **Fix 1 — Domain terms in `common_words`:** Removed "entity", "concept", "summary", "confidence", "speculative", "inferred" from `common_words` stoplist in `semantic.py` so pages sharing these KB-domain terms still get grouped for consistency checks.
- [x] **Fix 2 — Consistency group truncation:** `build_consistency_context()` now chunks page lists into sub-groups of `MAX_CONSISTENCY_GROUP_SIZE` instead of silently discarding pages beyond the limit.
- [x] **Fix 3 — Context truncation logging:** `_build_query_context()` in `engine.py` logs a `DEBUG` message when a page is excluded because remaining space is ≤ 100 chars.
- [x] **Fix 4 — BM25 avgdl guard:** `BM25Index` logs a `WARNING` when `avgdl` falls back to 1.0 due to an all-empty corpus.
- [x] **Fix 5 — Case-insensitive page lookup path validation:** `kb_read_page` now validates the resolved path stays within `WIKI_DIR` after case-insensitive fallback resolution.
- [x] **Fix 6 — `logger.exception` → `logger.error` in health tools:** `kb_lint` and `kb_evolve` use `logger.error` instead of `logger.exception` to avoid leaking stack traces to MCP clients.

### Group 3: Input Validation + Config
**Files:** `src/kb/feedback/store.py`, `src/kb/lint/verdicts.py`, `src/kb/review/refiner.py`, `pyproject.toml`
**Test:** `tests/test_validation_fixes_v092.py` (11 tests)

- [x] **Fix 1 — Feedback input validation:** `add_feedback_entry()` enforces: question/notes max 2000 chars, page ID max 200 chars, max 50 cited pages, path traversal rejection (`..` / leading `/`).
- [x] **Fix 2 — Verdict severity validation:** `add_verdict()` rejects unknown severity values; valid set is `("error", "warning", "info")`. Each issue must be a `dict`.
- [x] **Fix 3 — Frontmatter validation in refiner:** `refine_page()` rejects `new_content` that starts with `---` (prevents frontmatter corruption).
- [x] **Fix 4 — Dev dependencies:** `pytest` added to `[project.optional-dependencies]` in `pyproject.toml`.
