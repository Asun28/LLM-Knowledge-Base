### Step 4: Fix _feedback_lock — add sleep after eviction

- [ ] **4a: The sleep is already present at line 40**

Looking at the current code:

```python
        except FileExistsError:
            if time.monotonic() > deadline:
                # Stale lock — remove and retry acquisition
                lock_path.unlink(missing_ok=True)
                continue
            time.sleep(0.05)
```

The `time.sleep(0.05)` is only in the non-deadline branch. After eviction (deadline expired), the code does `unlink` then `continue` without sleeping — this is the CPU-spin issue. Add sleep after eviction:

```python

            if time.monotonic() > deadline:
                # Stale lock — remove and retry acquisition
                lock_path.unlink(missing_ok=True)
                continue
# NEW:
            if time.monotonic() > deadline:
                # Stale lock — remove and retry acquisition
                lock_path.unlink(missing_ok=True)

def get_coverage_gaps(path: Path | None = None) -> list[dict]:
    """Get questions where the answer was rated 'incomplete'.

    Deduplicates by question text, keeping the first occurrence.

    Returns:
        List of dicts with 'question' and 'notes' keys.
    """
    data = load_feedback(path)
    seen: set[str] = set()
    gaps = []
    for e in data.get("entries", []):
        if e.get("rating") == "incomplete" and e.get("question"):
            q = e["question"]
            if q not in seen:
                seen.add(q)
                gaps.append({"question": q, "notes": e.get("notes", "")})
    return gaps
```
