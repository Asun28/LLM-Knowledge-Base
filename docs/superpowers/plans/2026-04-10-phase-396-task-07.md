## Fix 7.1 — CRITICAL: Frontmatter guard regex empty block (`refiner.py:86`)

`---\n---` passes the `r"---\n.+\n---"` guard (requires ≥1 char between fences). LLM can return `---\n---\nReal body` which produces double-frontmatter, corrupting the page.

- [ ] **Step 1: Write test**

```python
class TestRefinerFrontmatterGuard:
    """Fix 7.1: refine_page must reject empty frontmatter blocks."""

    def test_empty_frontmatter_block_rejected(self, tmp_wiki, create_wiki_page):
        from kb.review.refiner import refine_page

        create_wiki_page("concepts/test", wiki_dir=tmp_wiki, content="Original.")
        result = refine_page(
            "concepts/test", "---\n---\nReal body", wiki_dir=tmp_wiki,
            history_path=tmp_wiki / "history.json",
        )
        assert "error" in result

    def test_normal_horizontal_rule_accepted(self, tmp_wiki, create_wiki_page):
        from kb.review.refiner import refine_page

        create_wiki_page("concepts/test2", wiki_dir=tmp_wiki, content="Original.")
        result = refine_page(
            "concepts/test2", "Updated content.\n\n---\n\nMore content.", wiki_dir=tmp_wiki,
            history_path=tmp_wiki / "history.json",
        )
        # Horizontal rules in content should be accepted
        assert "error" not in result
```

- [ ] **Step 2: Run test to verify it fails**

Run: `python -m pytest tests/test_v0915_task07.py::TestRefinerFrontmatterGuard -v`

- [ ] **Step 3: Implement fix in `refiner.py` line 86**

To:
