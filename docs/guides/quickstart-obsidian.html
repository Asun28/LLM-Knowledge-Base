Quickstart Guide

Fully Automated Knowledge Wiki
with Claude + Obsidian

Drop a source in. Everything else is automatic — Claude extracts entities, builds interlinked pages, injects wikilinks, tracks trust scores, and self-maintains. Browse it all in Obsidian.

raw/ sources → Claude compiles → wiki/ pages → Obsidian graph
★ Why This Exists

Compile, don't retrieve

Inspired by Karpathy's LLM Knowledge Bases — then fully automated. Karpathy described a pattern where you manually ask an LLM to compile pages. This system automates the entire pipeline.

🔄

RAG retrieves chunks

Vector embeddings you can't read. Top-K results with no relationship awareness. No way to know what's missing.

📖

This compiles knowledge

Markdown pages you can browse. Entities linked by wikilinks forming a graph. Lint catches contradictions. Evolve finds gaps.

Karpathy's pattern (manual)This project (fully automated)
Manually ask LLM to write pagesOne command → extraction, page creation, linking, indexing
No change detectionIncremental compile — SHA-256 hashes, only reprocesses what's new
No cross-linkingRetroactive wikilink injection into existing pages
No quality checksSelf-healing — lint, trust scoring, contradiction detection
No gap awarenessEvolve — automatically identifies missing coverage
Text-onlyObsidian — visual knowledge graph for free
1 Getting Started

Setup in 5 minutes

Prerequisites

  • Python 3.12+
  • Obsidian (free)
  • Claude Code or an Anthropic API key

Install

git clone https://github.com/Asun28/LLM-Knowledge-Base.git
cd LLM-Knowledge-Base
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
pip install -e .
kb --version
💡

Claude Code Max users: No API key needed. MCP tools use Claude Code as the LLM by default. Otherwise, copy .env.example to .env and add your ANTHROPIC_API_KEY.

2

Open folder as vault

Select the wiki/ directory inside the project. Not the project root.

3

Press Ctrl+G to see the graph

Every wiki page and its wikilinks are visualized instantly. Clusters show related concepts, orphan nodes show disconnected pages.

⚠️

Important: Open wiki/ as the vault, not the project root. This keeps raw sources and code out of Obsidian's view.

3 Try It

Your first ingest

One trigger. Everything else is automatic — extraction, page creation, cross-linking, indexing, and quality tracking.

Recommended

Claude Code (natural language)

Just tell Claude what to do:

"Ingest raw/articles/rag.md into my wiki"

Claude automatically: extracts entities & concepts, creates wiki pages with frontmatter, injects wikilinks into existing pages, updates all indexes, and flags affected pages for cascade review. Switch to Obsidian to see the graph grow.

CLI

Command line

# Grab an article
trafilatura -u URL > raw/articles/article.md

# Ingest it
kb ingest raw/articles/article.md --type article
One-shot

URL to wiki in one step

"Save this URL to my knowledge base:
 https://example.com/article"

Fetches, saves to raw/, and creates all wiki pages in a single call.

4 Fully Automated Cycle

The five operations

A continuous, self-maintaining feedback loop — each operation feeds into the next, all fully automated:

📥
Ingest
Extract entities, concepts, claims from raw sources
⚙️
Compile
Batch process all new or changed sources
🔍
Query
BM25 + PageRank search, synthesize answers
🧹
Lint
Dead links, orphans, staleness, frontmatter checks
🌱
Evolve
Gap analysis, missing coverage, connection suggestions
OperationCommandWhat to look for in Obsidian
Ingestkb ingest <file>New pages in entities/, concepts/, summaries/
Compilekb compileMultiple new pages from batch processing
Querykb query "..."Answer with citations to wiki page IDs
Lintkb lintOrphan nodes in graph (no connections)
Evolvekb evolveSuggested new pages and connections

6 Power User

Dataview query examples

Install the Dataview plugin, then create query pages in your vault:

Speculative claims

```dataview
TABLE source, updated
FROM ""
WHERE confidence = "speculative"
SORT updated DESC
```

Recent entities

```dataview
TABLE title, confidence, updated
FROM "entities"
SORT updated DESC
LIMIT 20
```

Pages from a source

```dataview
TABLE title, type
FROM ""
WHERE contains(source,
  "raw/articles/rag.md")
```

Stale pages

```dataview
TABLE title, updated
FROM ""
WHERE updated < date(today) - dur(90d)
SORT updated ASC
```
7 Workflow

Daily routine

Every day

Grow

  1. Find an interesting article, paper, or video
  2. Save to raw/ (trafilatura, crwl, yt-dlp)
  3. Ingest via Claude Code or CLI
  4. Open Obsidian — browse new pages, watch graph grow
Every week

Maintain

  1. kb lint — fix dead links, stale pages
  2. kb evolve — find gaps and missing connections
  3. Browse Obsidian graph for emerging clusters
  4. kb query — test cross-source synthesis
